Schema is composed from multiple `schema/*.prisma` fragments referenced by the root `schema.prisma`; running `prisma generate` consumes all of them into one client. Development migrations are created via `prisma migrate dev` and applied through the `migrations/` directory locked by `migration_lock.toml`. Seed entry points are plain Node scripts executed directly (e.g. `node prisma/seed.js`) rather than via a Prisma seed command.
