Two flat sub-folders with no shared library — each script is self-contained and invoked individually.
- `scripts/`: operational utilities. Sub-folder `management/` holds Windows PowerShell helpers for remote server checks (`check-all.ps1`, `debug-server.ps1`, `monitor-server.ps1`) that SSH into a Lightsail instance using a temporary key; the rest are single-purpose Node/TS scripts that import Excel (`xlsx`), patch QGIS assets (`patch-qgis-dynamic.py`, `repack-qgz.py`), reconcile users/serials via Prisma (`import-laptops.ts`, `relink-custodians.ts`, `verify-user-serials.ts`), or run integration tests against live API routes by importing their `POST`/`GET` handlers directly (`test-backend-agreement.ts`, `seed-handover-logs.ts`).
- `scratch/`: exploratory / ad-hoc SQL and endpoint probes (`check-db.ts`, `find-sods.js`, `scan-loops.ts`, `slt-ishamp-sync.user.js`); many files are transient experiments rather than maintained tooling.
Dependency direction is strictly inward: scripts depend on `src/lib/prisma`, `src/lib/redis`, and individual Next.js Route Handlers under `src/app/api/**`; nothing in `src/` imports back from these folders. There is no orchestrator — scripts are launched manually from the repo root.
