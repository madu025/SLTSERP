Three sibling top-level folders form the module:
- `QGIS Project Template/` — a reusable QGIS project (`QGIS.qgz`) plus an `Export/QGIS_qfield.qgs` variant configured for QField offline editing. The qfield project references a single local `data.gpkg` via OGR provider with `option:QGIS_FORCE_WAL=ON`, and ships a parallel `Export/GeoPackage/` directory containing one `.gpkg` per layer (SLT_TP, SLT_Poles, SLT_ODF, SLT_MH, SLT_HH, SLT_FTC, SLT_FJ, SLT_FDP, SLT_Road_EOPs, SLT_Risers, SLT_Ducts, SLT_Cables). A `QGIS_attachments.zip` / `QGIS_qfield_attachments.zip` holds field-collected media.
- `KL-SVK-0567/` and `SM-HJJ-0508-AB/` — per-project export bundles. Each contains its own `QGIS.qgz` plus subfolders named by output format (`CSV/`, `Shapes/`, `GeoJSON/`, `KMZ/`, and in KL also `Project_Data/DXF/`). Files are prefixed with the project code (e.g. `KL-SVK-0567_Cables.shp`, `SM-HJJ-0508_Cables.geojson`) so they can be dropped into any workspace without renaming.
Dependency direction is one-way: template projects consume the shared layer schema; exported bundles are leaf artifacts produced from those schemas and contain no code.
