Pure documentation artifact tree with no executable code — four sibling directories each serve a distinct audience:
- `docs/` holds end-user / operator manuals (`SYSTEM_DOCUMENTATION.md` covering architecture, deployment, QFieldCloud, SOD revenue config, vehicles) and the Stores Manager's bilingual (English/Sinhala) workflow guide (`STORES_WORKFLOW_GUIDE.md`) plus an archived Phoenix Bridge v4.5.0 zip.
- `memory/` is a flat, date-stamped journal (`YYYY-MM-DD.md`) where developers log UI refactors and verification results; files are appended chronologically rather than versioned by feature.
- `.agent/` is the LLM-agent playbook: `AGENTS.md` enforces production-level rules (Next.js route caching, OpenLayers sizing, DTOs, pagination), while `CODEMAP.md`, `DESIGN_SYSTEM.md`, `PAGE_BLUEPRINT_TEMPLATE.md`, `IMAGE_STORAGE_MIGRATION_ANALYSIS.md`, and `instructions.md` provide structural maps, design tokens, page templates, and migration analysis for agents to consult before writing code.
- `ai-tasks/` contains the Vehicle Management module backlog as a single JSON state file (`vehicle_management_ai_tasks.json`) consumed by an autonomous agent that picks the lowest-id `not-started` task whose dependencies are all `completed`, then claims/completes it per the embedded `agent_instructions.pick_policy` / `claim` / `complete` rules; `vehicle_management_ai_tasks.md` mirrors the same plan in Markdown.

Dependency direction is one-way: source code never imports these files; instead, agents read them to understand domain context, coding standards, and outstanding work before touching `src/`.
